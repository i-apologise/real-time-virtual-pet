# Needs your review (merge when ready)

## Why you were not notified

Agents open PRs with **your** GitHub account via `gh`. GitHub treats that as **your own activity** and usually **does not send you** a “new PR” email/notification (same as opening a PR in the browser while logged in as yourself).

This is a **GitHub behavior**, not a bug in the agent.

## How to get notified going forward

1. **Repo → Watch → All activity** (or Custom → Pull requests + Issues)  
   https://github.com/i-apologise/real-time-virtual-pet  
2. Check **[Pull requests](https://github.com/i-apologise/real-time-virtual-pet/pulls)** regularly.
3. Check this file: `docs/NEEDS-YOUR-REVIEW.md` (agents must update it when they open a PR).
4. The **15m watchdog** will keep listing open PRs in its status when it runs in chat.
